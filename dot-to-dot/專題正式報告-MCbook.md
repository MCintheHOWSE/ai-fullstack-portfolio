摘要
　　隨著共享經濟商業模式的成熟，利用數位平台解決特定場域內的供需失衡已成為趨勢。東吳大學受限於外雙溪與城中雙校區的地理阻隔，學生長期面臨跨校區通勤困難、宿舍搬遷成本高昂以及校園餐飲選擇受限等問題。雖然市面上有 Uber 或 Lalamove 等商業服務，但其高昂定價與無法深入校園場景的限制，使得學生需求無法被有效滿足。

　　因此，本專題旨在設計並實作一套名為「Dot to Dot」的校園智慧互助平台。我們利用 React 前端框架與 Node.js 後端技術，結合 WebSocket 即時通訊與零知識支付架構，將校園內閒置的運能（機車後座、汽車空間）與勞力轉化為可被利用的資源。本系統整合了「共乘媒合」、「校園物流」與「跑腿服務」三大功能，透過數位化手段重建校園熟人社會的信任機制，解決學生的剛性需求，並驗證了校園共享經濟的可行性。

第一章、緒論(含動機、需求分析)
第一節、研究背景與動機
　　我們這次的專案主題是「針對大學生的智慧共乘平台設計與實作」。隨著共享經濟的概念普及，我們觀察到校園內存在大量的「閒置運能」，例如騎機車往返雙校區的同學，其後座空間往往是閒置的；又或是週末返家的休旅車，具有載運大型行李的能力。 　　東吳大學受限於外雙溪與城中校區的地理距離，學生依賴公車（557、300路）或校車，但在尖峰時刻往往一位難求。此外，每學年的宿舍搬遷潮以及日常的餐飲購買，都因為地形與交通因素而產生高昂的成本。 　　因此，本研究動機在於透過開發一套專屬東吳人的 App，將這些閒置資源與剛性需求進行媒合，建立一個互助、低成本且高信任度的校園生活圈。

第二節、需求痛點分析
　　針對東吳大學的特殊場域，我們分析出以下三大痛點：

交通通勤痛點：兩校區距離約 30 分鐘車程，公車班次在離峰時段過少，尖峰時段過於擁擠。計程車單趟成本約 300 元，對無收入的學生負擔過重。

物流搬遷痛點：宿舍與校外租屋處分散，學生期末搬家時面臨「東西太少不值得叫搬家公司，東西太多無法手提」的窘境。

餐飲獲取痛點：外雙溪校區周邊餐飲選擇較少，商業外送平台運費高昂且有低消限制。然而，每天都有大量同學往返士林夜市或城中商圈，這些順路的行程具有極高的代購價值。

第三節、三大核心服務
　　基於上述分析，Dot to Dot 平台設計了三大核心服務模組：

共乘媒合 (Ride Sharing)：採用「駕駛主導」模式，由乘客發布需求，駕駛挑選順路訂單，解決通勤問題。

校園物流 (Logistics)：將「運力」與「勞力」解構，提供純運送、純搬運或全包式搬家服務，解決宿舍搬遷問題。

跑腿服務 (Food & Errands)：透過「順路開團」與「專屬許願」雙模式，結合實報實銷機制，解決校園餐飲獲取問題。

第二章、研究方法(或步驟)
第一節、System Architecture (系統架構說明)
　　本系統採用現代化的前後端分離架構 (Client-Server Architecture)。前端作為單頁式應用 (SPA) 負責使用者互動與介面渲染，後端負責業務邏輯處理與資料庫存取，兩者透過 RESTful API 與 WebSocket 進行溝通。

前端 (Client)：使用 React 框架開發，採用 Mobile-First 設計原則，確保在手機瀏覽器上的操作體驗。

後端 (Server)：使用 Node.js 執行環境搭配 Express 框架，處理 API 請求。

即時通訊：整合 Socket.io 服務，處理聊天室訊息與訂單狀態的即時推播。

第二節、技術規格與實作方法
　　為了實現各項功能，我們使用了以下關鍵技術：

前端技術：採用 React 19 與 Vite 進行開發。在介面設計上，捨棄複雜的地圖 API，改採 POI (Point of Interest) 預定義地點選擇器，以解決 GPS 定位漂移與隱私疑慮。

後端技術：使用 Node.js 與 Express。在物品量化上，後端實作了 JSON 解析邏輯，將前端傳來的 {box: 2, chair: 1} 結構化數據轉換為運費計算的依據。

零知識支付 (Zero-Knowledge Payment)：為了規避金流法規風險與資安疑慮，本系統採用用戶端加密暫存 (Client-Side Storage) 技術。使用者的銀行帳號僅儲存於手機 LocalStorage，交易時透過 WebSocket 點對點傳輸，伺服器端不留存任何金融敏感資訊。

第三節、資料庫設計與隱私權保護
　　本系統使用 SQLite 關聯式資料庫，並遵循正規化原則設計。針對隱私保護，我們制定了嚴格的資料治理條款：

資料最小化：僅蒐集必要的學號信箱與行程起訖點。

閱後即焚機制：訂單完成後，前台立即遮蔽雙方個資，後台資料庫於 7 天緩衝期後執行物理刪除 (Hard Delete)，保障用戶隱私。

第四章、結論(含心得)
第一節、現有結論
　　本專題成功開發出「Dot to Dot」校園智慧互助平台，透過實際的系統設計與開發，我們得出以下結論：

運力活化的可行性：透過將「運力」與「勞力」解構，我們證實了校園內確實存在大量可被共享的閒置資源。不論是機車後座或是週末閒置的體力，都能轉化為服務價值。

信任機制的建立：雖然去除了金流託管，但透過「學號驗證」、「現場核對」與「雙向評分」機制，我們在數位平台上成功重現了校園熟人社會的信任感。

技術與隱私的平衡：採用的「零知識支付」架構，成功在不觸碰金融法規紅線的前提下，提供了便捷的轉帳體驗，並最大程度保障了使用者的資安隱私。

第二節、發想(未來的應用)
　　針對本系統的未來發展，我們提出以下發想：

演算法優化：目前系統採用 POI 定點媒合，未來可引入圖論演算法，針對「順路程度」進行更精確的權重計算，自動推薦最佳的共乘或跑腿路徑。

跨校區聯盟：本系統架構具備可複製性，未來可推廣至地理位置相近的大學聯盟（如東吳、銘傳、文化），擴大共享經濟的網絡效應。

商業模式轉型：目前平台定位為校園公共財，未來若用戶基數擴大，可考慮引入周邊商家的廣告推播（如餐廳優惠券），建立可持續的營運模式。

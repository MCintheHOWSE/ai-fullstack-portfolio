# Dot to Dot 校園服務系統 - 完整測試指南

## 📖 系統概述

**Dot to Dot** 是一個整合性校園服務平台，提供以下服務：
- 🚗 **共乘系統** (Ride-sharing)
- 🏃 **跑腿服務** (Errands)
- 🚚 **物流搬運** (Logistics)
- 🍔 **美食外送/團購** (Food Delivery/Group Buy)
- 👨‍💼 **管理員後台** (Admin Dashboard)

---

## 🚀 系統啟動

### 前置需求
- Node.js 已安裝
- npm 套件已安裝

### 啟動步驟

1. **啟動後端伺服器**
```bash
cd 大四專題報告
node server/index.js
```
✅ 確認訊息：`Server running on http://localhost:3000`

2. **啟動前端開發伺服器**（新終端視窗）
```bash
cd 大四專題報告
npm run dev
```
✅ 確認訊息：`Local: http://localhost:5173/`

3. **打開瀏覽器**
- 訪問: http://localhost:5173

### 5.3 **實報實銷流程**（核心創新功能）⭐

> **這是美食外送的最大亮點**：解決「預估vs實際」差價問題

**前置：** 團購已有至少2人加入

**測試步驟：**

**階段1：跑腿者停止收單**
1. 團長點擊「停止收單」
2. 狀態變更為「準備購買」
3. 跑腿者前往餐廳

**階段2：輸入實際金額（關鍵步驟）**
4. 跑腿者回到訂單詳情頁
5. 找到「輸入實際金額」區塊
6. 輸入發票上的真實金額：
   - 範例：原預估 130，實際 140（多了一份加大薯條）
7. 點擊「確認實際金額」

**驗證點（實報實銷邏輯）：**
- ✅ 系統自動計算：實際總額 = $140 + 運費
- ✅ **所有跟團者立即收到推播通知**
- ✅ 通知內容：「實際金額已更新為 NT$140，請準備付款」
- ✅ 訂單顯示「實際金額」與「預估金額」對比
- ✅ 差額透明顯示（±10元）

**階段3：付款確認**
8. 跟團者查看更新的金額
9. 確認付款（使用零知識支付卡片）
10. 跑腿者確認收款

**Demo 話術建議：**
> 「這裡可以看到，當跑腿者輸入實際金額140元後，系統會透過 Socket.io **毫秒級別**推播給所有跟團者。這解決了傳統團購『到底要付多少錢』的爭議問題，做到完全透明的實報實銷。」

## 🐛 常見問題排查

### 問題 1：無法註冊

**症狀：** Email 驗證失敗  
**原因：** 必須使用 @scu.edu.tw 結尾的信箱  
**解決：** 確認信箱格式正確

---

### 問題 2：Socket.io 通知不工作

**症狀：** 沒有收到實時通知  
**檢查：**
1. 開發者工具 → Network → WS（WebSocket）
2. 確認有 Socket.io 連接
3. 兩個視窗都已登入

**解決：**
- 重新整理頁面
- 確認後端伺服器運行中

---

### 問題 3：API 請求失敗

**症狀：** 按鈕點擊無反應或錯誤  
**檢查：**
1. F12 開發者工具 → Console
2. Network → Fetch/XHR
3. 查看具體錯誤訊息

**常見原因：**
- 後端未啟動（確認 localhost:3000）
- CORS 問題
- 資料庫錯誤

---

### 問題 4：付款資訊未顯示

**症狀：** PaymentCard 沒有出現  
**檢查：**
- 確認訂單狀態正確
- 檢查 Socket.io 連接
- 確認雙方都在聊天室

---

### 問題 5：Modal 無法關閉

**解決：**
- 點擊 Modal 外部區域
- 按 ESC 鍵
- 重新整理頁面
